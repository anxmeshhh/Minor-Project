Research paper gaps — what VitalGlove fills
G1
No real-time end-to-end deployment — Most papers (PMC 2023, arXiv 2025) show models in lab settings, not deployed on real hardware. VitalGlove runs TinyML on ESP32 with live BLE transmission to a real app.
G2
No patient-facing interface — Reviewed systems (ScienceDirect, IJCA) only alert doctors. VitalGlove gives patients their own live dashboard, risk score, and history — making it truly patient-centric.
G3
Scalability not addressed — Papers flag device heterogeneity (ScienceDirect 2025). VitalGlove uses a standardized BLE + Firebase pipeline that scales across patient profiles via cloud functions.
G4
No emergency escalation layer — All reviewed papers stop at "alert generation." VitalGlove adds tiered escalation: doctor → family → ambulance, filling a critical safety gap.
G5
Data privacy left as future work — IoT/AI papers (ScienceDirect) cite privacy as unsolved. VitalGlove addresses this with encrypted Firebase storage + role-based access (patient vs doctor vs emergency).
G6
No cloud + edge hybrid — Papers use either edge or cloud, not both. VitalGlove combines edge inference (TinyML on ESP32) for speed with cloud analytics for deep trend analysis — a dual-layer approach.
G7
EHR/doctor integration missing — Springer mHealth study notes poor integration with healthcare systems. VitalGlove's doctor dashboard + messaging + alert interface directly bridges this gap.
